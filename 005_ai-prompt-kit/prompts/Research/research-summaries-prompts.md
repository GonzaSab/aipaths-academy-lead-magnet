# Research & Summary Prompts

> Free prompts for research synthesis, summarization, and knowledge extraction. Sources: [Founderpath](https://founderpath.com/blog/top-ai-business-prompts), [ClickUp](https://clickup.com/blog/ai-prompt-templates/), [Juma](https://juma.ai/blog/ai-prompts-for-business)

---

## Document Summarization

### Executive Summary from Report
```
Create an executive summary of this report/document:

[PASTE DOCUMENT OR KEY SECTIONS]

Summary requirements:
- Length: [100/250/500] words
- Audience: [EXECUTIVES/TEAM/CLIENTS]
- Focus: [DECISIONS/INSIGHTS/ACTIONS]

Include:
- Key findings (3-5 bullets)
- Critical data points
- Recommendations
- Next steps
- Caveats or limitations
```

### Meeting Notes to Action Items
```
Convert these meeting notes into structured action items:

Meeting: [MEETING NAME]
Date: [DATE]
Attendees: [NAMES/ROLES]

Notes:
[PASTE RAW NOTES]

Extract:
- Decisions made (with context)
- Action items (owner, deadline, details)
- Open questions/parking lot
- Key discussion points
- Follow-up meeting needs
```

### Long Document Breakdown
```
Summarize this long document in a hierarchical format:

[PASTE DOCUMENT]

Provide:
1. One-sentence overview
2. Key themes (3-5)
3. Summary of each major section
4. Important quotes/data points
5. Questions this raises
6. Relationship to [CONTEXT/PROJECT]

Keep total summary under [X] words.
```

## Research Synthesis

### Multi-Source Research Synthesis
```
Synthesize findings from these sources about [TOPIC]:

Source 1: [SUMMARY/KEY POINTS]
Source 2: [SUMMARY/KEY POINTS]
Source 3: [SUMMARY/KEY POINTS]

Provide:
- Common themes across sources
- Conflicting findings
- Gaps in the research
- Strongest supported conclusions
- Areas needing more research
- Practical implications

Weight credibility of sources: [HIGH/MEDIUM/LOW for each]
```

### Literature Review Structure
```
Help me structure a literature review on [TOPIC].

Research question: [QUESTION]
Sources collected: [NUMBER]
Time period covered: [DATE RANGE]

Key themes emerging:
- [THEME 1]
- [THEME 2]
- [THEME 3]

Create:
- Organizational framework
- Section outline
- Transition logic between sections
- Critical analysis prompts
- Gap identification framework
- Synthesis approach
```

### Trend Analysis from Articles
```
Analyze these articles/reports to identify trends in [INDUSTRY/TOPIC]:

Sources:
1. [TITLE] - [DATE] - [KEY POINT]
2. [TITLE] - [DATE] - [KEY POINT]
3. [Continue...]

Identify:
- Emerging trends (last 6-12 months)
- Established trends continuing
- Declining trends
- Contradictory signals
- Predictions for next [PERIOD]
- Implications for [MY BUSINESS/PROJECT]
```

## Information Extraction

### Key Facts Extraction
```
Extract key facts and figures from this content:

[PASTE CONTENT]

Organize into:
- Quantitative data (numbers, percentages, dates)
- Qualitative insights (opinions, predictions)
- Named entities (people, companies, products)
- Relationships (X causes Y, A acquired B)
- Timelines (sequence of events)

Flag any uncertain or conflicting information.
```

### Technical Documentation Summary
```
Summarize this technical documentation for [AUDIENCE: developers/managers/clients]:

[PASTE DOCUMENTATION]

Create:
- Overview (what this does)
- Key capabilities
- Limitations/constraints
- Prerequisites
- Quick start guide (condensed)
- Common use cases
- Gotchas/warnings

Technical level: [BEGINNER/INTERMEDIATE/EXPERT]
```

### Comparison Extraction
```
Extract a comparison from this content:

[PASTE CONTENT COMPARING OPTIONS/PRODUCTS/APPROACHES]

Create a structured comparison:
- Options being compared
- Comparison criteria
- Scoring/evaluation for each
- Pros and cons
- Use case recommendations
- Source's recommendation (if any)
- Missing considerations

Format as a table where possible.
```

## Knowledge Base Creation

### FAQ Generation from Content
```
Generate a FAQ document from this content:

[PASTE CONTENT: product docs, support tickets, meeting notes, etc.]

Create [X] Q&A pairs covering:
- Basic understanding questions
- How-to questions
- Troubleshooting questions
- Best practices
- Edge cases

For each answer:
- Clear, concise response
- Link to source section
- Related questions
```

### Process Documentation from Notes
```
Convert these informal notes into structured documentation:

[PASTE NOTES, SLACK MESSAGES, EMAILS about a process]

Create:
- Process name and purpose
- Step-by-step instructions
- Decision points
- Roles and responsibilities
- Tools/systems used
- Tips from experience
- Common mistakes to avoid

Format for [WIKI/CONFLUENCE/NOTION]
```

### Glossary Builder
```
Create a glossary from this content for [AUDIENCE]:

[PASTE CONTENT with domain-specific terms]

For each term:
- Simple definition
- Extended explanation
- Example in context
- Related terms
- Common misconceptions

Organize alphabetically.
Indicate technical level: [BASIC/ADVANCED]
```

## Report Generation

### Research Report Template
```
Create a research report structure for [TOPIC].

Research objective: [OBJECTIVE]
Audience: [WHO WILL READ]
Length: [PAGE COUNT]

Sections needed:
- Executive summary
- Methodology
- Findings
- Analysis
- Recommendations
- Appendices

For each section provide:
- Purpose
- Key questions to answer
- Data/evidence to include
- Length guidance
- Formatting tips
```

### Weekly/Monthly Report Automation
```
Create a template for automating my [WEEKLY/MONTHLY] [REPORT TYPE] report.

Stakeholders: [AUDIENCE]
Data sources: [WHERE DATA COMES FROM]

Current manual process:
[DESCRIBE CURRENT STEPS]

Create:
- Report structure template
- Metrics to auto-populate
- Commentary prompts (fill-in-the-blank)
- Visualization recommendations
- Distribution checklist
- Archive naming convention
```

### Client-Ready Summary
```
Transform this internal analysis into a client-facing summary:

Internal content:
[PASTE INTERNAL DOCUMENT]

Client: [CLIENT TYPE/INDUSTRY]
Relationship: [NEW/EXISTING]
Purpose: [INFORM/PERSUADE/REPORT]

Adjust for:
- Remove internal jargon
- Add context they need
- Highlight relevance to them
- Appropriate level of detail
- Professional tone
- Clear next steps/CTA

Length: [WORD COUNT]
```

## Quick Prompts

### TLDR Generator
```
Give me a TLDR of this in [1-3] sentences:

[PASTE CONTENT]

Focus on: [KEY ASPECT]
Audience context: [WHO NEEDS THIS]
```

### Bullet Point Summary
```
Summarize this in [X] bullet points:

[PASTE CONTENT]

Each bullet should:
- Be self-contained
- Start with action verb or key noun
- Be under 20 words
- Cover a distinct point
```

### Implications Analysis
```
What are the implications of this information?

[PASTE CONTENT]

Analyze implications for:
- [STAKEHOLDER 1]
- [STAKEHOLDER 2]
- [MY ROLE/BUSINESS]

Include:
- Immediate implications
- Long-term implications
- Risks
- Opportunities
- Required actions
```
