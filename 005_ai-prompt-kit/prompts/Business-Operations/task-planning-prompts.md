# Task Planning & Project Management Prompts

> Free prompts for task breakdown, planning, and productivity. Sources: [ClickUp](https://clickup.com/p/ai-prompts/strategic-business-analysis), [Learn Prompting](https://learnprompting.org/blog/ai-prompt-business-planning-strategy), [Juma](https://juma.ai/blog/ai-prompts-for-business)

## Task Breakdown

### Complex Task Decomposition
```
Break down this project into actionable tasks:

Project: [PROJECT NAME]
Goal: [END OBJECTIVE]
Timeline: [DEADLINE OR DURATION]
Team size: [NUMBER OF PEOPLE]
Constraints: [BUDGET, RESOURCES, DEPENDENCIES]

Provide:
1. Major milestones (3-5)
2. Tasks under each milestone
3. Dependencies between tasks
4. Estimated effort (S/M/L) for each task
5. Suggested sequence/parallel tracks
6. Risk items to monitor

Format as a structured task list I can import into [PROJECT TOOL].
```

### Sprint Planning Assistant
```
Help me plan a [X]-week sprint for [TEAM/PROJECT].

Goals for this sprint:
- [GOAL 1]
- [GOAL 2]
- [GOAL 3]

Team capacity: [X] story points / [Y] hours
Carryover items: [LIST IF ANY]

Available work:
- [FEATURE/TASK 1]: [ESTIMATED EFFORT]
- [FEATURE/TASK 2]: [ESTIMATED EFFORT]
- [Continue...]

Provide:
- Recommended sprint scope
- Task prioritization (MoSCoW)
- Risk assessment
- Dependencies to resolve first
- Success criteria for sprint
```

### Daily/Weekly Planning
```
Help me plan my [day/week] based on these inputs:

Priorities:
1. [MUST DO 1]
2. [MUST DO 2]
3. [SHOULD DO 1]

Meetings/commitments:
- [TIME]: [COMMITMENT]
- [TIME]: [COMMITMENT]

Available focus time: [HOURS]
Energy pattern: [morning person/afternoon peak/etc.]

Create:
- Time-blocked schedule
- Task batching suggestions
- Buffer time allocation
- Realistic task limits
- End-of-day review checklist
```

## Strategic Planning

### OKR Framework
```
Help me create OKRs for [TEAM/DEPARTMENT/COMPANY].

Time period: [QUARTER/YEAR]
Company mission: [MISSION]
Key focus areas: [AREAS]

Current challenges:
- [CHALLENGE 1]
- [CHALLENGE 2]

For each objective, provide:
- Objective statement (qualitative, inspirational)
- 3-4 Key Results (quantitative, measurable)
- Leading indicators
- Initiatives that support each KR
- Potential obstacles

Format for [COMPANY SIZE] organization.
```

### Roadmap Creation
```
Create a product/project roadmap for [PROJECT/PRODUCT].

Vision: [LONG-TERM VISION]
Time horizon: [6 months/1 year/3 years]

Current state:
- [WHERE WE ARE NOW]

Key initiatives:
- [INITIATIVE 1]
- [INITIATIVE 2]
- [INITIATIVE 3]

Constraints:
- Budget: [BUDGET]
- Team: [SIZE]
- Dependencies: [EXTERNAL DEPS]

Provide:
- Phased approach with themes per phase
- Key milestones
- Dependencies visualization
- Resource allocation suggestions
- Go/no-go decision points
```

## Meeting & Communication

### Meeting Agenda Generator
```
Create an agenda for [MEETING TYPE] meeting.

Purpose: [GOAL OF MEETING]
Duration: [TIME]
Attendees: [ROLES/PEOPLE]

Topics to cover:
- [TOPIC 1]
- [TOPIC 2]
- [TOPIC 3]

Provide:
- Timed agenda with sections
- Discussion questions for each topic
- Pre-meeting preparation items
- Expected outcomes/decisions
- Action item template
- Follow-up structure
```

### Status Update Template
```
Create a status update template for [PROJECT/TEAM].

Audience: [STAKEHOLDERS]
Frequency: [DAILY/WEEKLY/MONTHLY]

Include sections for:
- Overall health (RAG status)
- Key accomplishments
- Upcoming priorities
- Blockers/risks
- Metrics/KPIs
- Asks/needs

Keep it [BRIEF/DETAILED] - approximately [LENGTH].
Include prompting questions to fill each section.
```

### Decision Document Framework
```
Create a decision document for: [DECISION TO MAKE]

Context: [BACKGROUND]
Stakeholders: [WHO'S INVOLVED]
Timeline: [WHEN DECISION NEEDED]

Options being considered:
1. [OPTION 1]
2. [OPTION 2]
3. [OPTION 3]

Structure the document:
- Problem statement
- Decision criteria (weighted)
- Options analysis
- Pros/cons matrix
- Recommendation
- Implementation plan
- Reversibility assessment
```

## Process Optimization

### Workflow Analysis
```
Analyze this workflow and suggest improvements:

Current process:
1. [STEP 1] - [WHO] - [TIME]
2. [STEP 2] - [WHO] - [TIME]
3. [STEP 3] - [WHO] - [TIME]
[Continue...]

Pain points:
- [ISSUE 1]
- [ISSUE 2]

Goals:
- Reduce time by [X]%
- Reduce errors
- Improve handoffs

Provide:
- Bottleneck identification
- Automation opportunities
- Parallel processing options
- Elimination candidates (non-value-add steps)
- Redesigned workflow
```

### SOP Creation
```
Create a Standard Operating Procedure for [PROCESS].

Process name: [NAME]
Purpose: [WHY THIS EXISTS]
Frequency: [HOW OFTEN]
Responsible: [ROLE]

Key steps (high level):
1. [STEP 1]
2. [STEP 2]
3. [STEP 3]

Provide:
- Detailed step-by-step instructions
- Decision points with criteria
- Quality checkpoints
- Common errors and prevention
- Escalation procedures
- Related documents/tools
- Version control section
```

### Capacity Planning
```
Help me plan capacity for [TEAM/RESOURCE].

Current state:
- Team size: [NUMBER]
- Current utilization: [%]
- Skill distribution: [SKILLS]

Upcoming demand:
- [PROJECT 1]: [EFFORT ESTIMATE]
- [PROJECT 2]: [EFFORT ESTIMATE]
- [PROJECT 3]: [EFFORT ESTIMATE]

Constraints:
- Budget for hiring: [AMOUNT]
- Ramp-up time: [WEEKS]
- Attrition risk: [%]

Provide:
- Capacity gap analysis
- Hiring vs outsourcing recommendation
- Skill gap identification
- Prioritization framework for work
- Buffer recommendations
```

## Research & Analysis

### Research Brief Template
```
Create a research brief for investigating [TOPIC].

Research question: [MAIN QUESTION]
Business context: [WHY THIS MATTERS]
Decision this will inform: [DECISION]

Scope:
- In scope: [WHAT TO COVER]
- Out of scope: [WHAT TO EXCLUDE]

Provide:
- Research methodology recommendation
- Key sub-questions to answer
- Data sources to explore
- Expert interviews to conduct
- Timeline estimate
- Deliverable format
```
